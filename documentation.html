Налаштування інтеграції Google Calendar з Telegram

Ця документація допоможе адміністраторам швидко налаштувати пересилку подій з Google Календаря у вибрані Telegram-групи.

1. Отримання ID календаря

Щоб система розуміла, який календар слухати, вам потрібно знайти його унікальний ідентифікатор (ID).

  1. Відкрийте Google Календар на комп'ютері.
  2. На лівій панелі знайдіть потрібний календар, натисніть на три крапки (⋮) поруч із ним та виберіть Налаштування та доступ.
  3. Прокрутіть сторінку вниз до розділу Інтеграція календаря.
  4. Скопіюйте та збережіть собі Ідентифікатор календаря. Зазвичай він виглядає як email (наприклад: ...random_string...@group.calendar.google.com).

2. Налаштування Google Apps Script

Цей скрипт буде автоматично відправляти інформацію на наш сервер кожного разу, коли в календарі створюється, змінюється або видаляється подія.

Обов'язковий крок: Оскільки наш скрипт працює з Advanced Calendar Service, перед написанням коду потрібно активувати Google Calendar API!
  1. Перейдіть на сайт Google Apps Script.
  2. Натисніть кнопку Новий проєкт.
  3. На лівій навігаційній панелі знайдіть розділ Сервіси (Services) та натисніть на плюсик (+) поруч з ним.
  4. У списку знайдіть та оберіть Google Calendar API, після чого натисніть Додати (Add).
  5. Тепер у центральному редакторі видаліть весь стандартний код і вставте код, наведений нижче.
  6. У рядку const CALENDAR_IDS = [...] вставте ID вашого календаря (або кількох календарів), який ви скопіювали на попередньому кроці(пункт 1.4).
  7. Натисніть значок дискети (Зберегти) або Ctrl+S.

Код, який потрібно вставити на 5 кроці:

const WEBHOOK_URL = 'https://calendar-forwarder-production.up.railway.app/calendar-webhook'; 

const CALENDAR_IDS = [
  'b92fub1u6r4em0h53qda95cp6k@group.calendar.google.com'
]; 

function setupTrigger() {
  const triggers = ScriptApp.getProjectTriggers();
  for (let i = 0; i < triggers.length; i++) {
    ScriptApp.deleteTrigger(triggers[i]);
  }
  
  PropertiesService.getScriptProperties().deleteAllProperties();
  
  CALENDAR_IDS.forEach(calendarId => {
    try {
      ScriptApp.newTrigger('onCalendarSync')
        .forUserCalendar(calendarId)
        .onEventUpdated()
        .create();
        
      fetchAndSaveSyncToken(calendarId);
    } catch (e) {
      console.error('Помилка створення тригера для календаря ' + calendarId + ': ' + e.message);
    }
  });
}

function fetchAndSaveSyncToken(calendarId) {
  try {
    const calendar = Calendar.Events.list(calendarId, {
      timeMin: (new Date()).toISOString()
    });
    PropertiesService.getScriptProperties().setProperty('syncToken_' + calendarId, calendar.nextSyncToken);
  } catch (e) {
    console.error('Помилка отримання токена для ' + calendarId + ': ' + e.message);
  }
}

function onCalendarSync(e) {
  const currentCalendarId = e.calendarId; 
  if (!currentCalendarId) return;

  const properties = PropertiesService.getScriptProperties();
  const tokenKey = 'syncToken_' + currentCalendarId;
  const syncToken = properties.getProperty(tokenKey);
  
  let options = { showDeleted: true };
  if (syncToken) {
    options.syncToken = syncToken;
  } else {
    options.timeMin = (new Date()).toISOString();
  }
  
  const cal = CalendarApp.getCalendarById(currentCalendarId);
  const calendarDefaultHex = cal ? cal.getColor() : '#039be5';
  const calendarTimeZone = cal ? cal.getTimeZone() : 'Europe/Kyiv';
  
  try {
    const events = Calendar.Events.list(currentCalendarId, options);
    events.items.forEach(item => {
      let status = (item.status === 'cancelled') ? 'deleted' : (item.created === item.updated ? 'created' : 'updated');
      
      const payload = {
        calendarId: currentCalendarId,
        eventId: item.id,
        status: status,
        title: item.summary || 'Без назви',
        start: item.start,
        end: item.end,
        calendarTimeZone: calendarTimeZone,
        colorId: item.colorId || calendarDefaultHex,
        creatorEmail: (item.creator && item.creator.email) ? item.creator.email : 'невідомо',
        eventLink: item.htmlLink || ''
      };
      
      UrlFetchApp.fetch(WEBHOOK_URL, {
        method: 'post',
        contentType: 'application/json',
        payload: JSON.stringify(payload),
        muteHttpExceptions: true
      });
    });
    
    if (events.nextSyncToken) {
      properties.setProperty(tokenKey, events.nextSyncToken);
    }
  } catch (error) {
    properties.deleteProperty(tokenKey);
    fetchAndSaveSyncToken(currentCalendarId);
  }
}

3. Авторизація та Запуск

Щоб скрипт запрацював, його потрібно активувати та надати дозволи на читання вашого календаря.

  1. На верхній панелі у випадаючому списку виберіть функцію setupTrigger (замість onCalendarSync).
  2. Натисніть кнопку Виконати (Run).
  3. З'явиться вікно «Потрібна авторизація». Натисніть «Переглянути дозволи».
  4. Виберіть свій Google-акаунт.
    Якщо з'явиться попередження про неперевірений додаток, натисніть Додатково (Advanced) -> Перейти до проєкту (Go to project).
  5. Натисніть Дозволити (Allow).
✅ Готово! З боку Google Календаря все налаштовано. Дані вже відправляються на сервер.

4. Додавання бота в групу Telegram

Щоб отримувати сповіщення, вам потрібно додати нашого бота у цільову групу і надати йому права адміністратора.

  1. Відкрийте Telegram групу, куди хочете отримувати сповіщення.
  2. Зайдіть у налаштування групи -> Додати учасника.
  3. Знайдіть бота за його username: @CalendarForwarderBot
  4. Зайдіть у Редагування -> Адміністратори та додати Адміністратора.
  5. Дайте йому права на відправку та редагування повідомлень

5. Прив'язка календаря (/bind)

Наш сервер вже отримує події, але йому треба вказати, в який саме чат їх відправляти. Для цього використовується команда прив'язки.

  1. У вашій групі в Telegram напишіть повідомлення з текстом, який починається з команди /bind, після якої через пробіл вкажіть ID календаря, який ви отримали раніше(пункт 1.4).:
    /bind b92fub1u6r4em0h53qda95cp6k@group.calendar.google.com
  2. Бот миттєво видалить ваше повідомлення (щоб ніхто в групі не побачив ID календаря) і відповість: ✅ Календар успішно прив'язано до цієї групи!

    Це все! Налаштування завершено! Додайте подію у ваш календар, і вона автоматично з'явиться в групі.

Важливо: Один і той самий календар не може бути прив'язаний до кількох різних груп одночасно. Це зроблено для безпеки. Якщо вам потрібно перенести календар в іншу групу, спочатку відв'яжіть його в старій групі.

Додаткові команди (тільки для адміністраторів):

Команда Опис
/unbind <calendar_id> Відв'язує конкретний календар від поточної групи.
/unbindall Відв'язує абсолютно всі календарі від поточної групи.
